# 故障排除和报告问题

如果您在使用 Rayforge 时遇到问题，特别是连接或控制您的机器方面，我们很乐意为您提供帮助。获得支持的最佳方式是提供详细的调试报告。Rayforge 有一个内置工具可以简化此过程。

## 如何创建调试报告

按照以下简单步骤生成并分享报告：

#### 1. 保存报告

转到菜单栏中的 **帮助 → 保存调试日志**。这会将所有必要的诊断信息打包到单个 `.zip` 文件中。将此文件保存到容易记住的位置，如桌面。

#### 2. 创建 GitHub Issue

转到我们的 [GitHub Issues 页面](https://github.com/barebaric/rayforge/issues/new/choose) 并创建新 issue。请提供清晰的标题和详细的问题描述：

- **您做了什么？**（例如，"我在启动应用程序后尝试连接到我的激光。"）
- **您期望发生什么？**（例如，"我期望它能成功连接。"）
- **实际发生了什么？**（例如，"它保持断开状态，日志显示超时错误。"）

#### 3. 附加报告

将您保存的 `.zip` 文件 **拖放到** GitHub issue 的描述框中。这将上传并将其附加到您的报告。

## 调试报告中包含什么

生成的 `.zip` 文件包含帮助我们快速诊断问题的技术信息。它包括：

- **机器和应用程序设置：** 您保存的机器配置和应用程序首选项，帮助我们重现您的设置。
- **自定义方言：** 您创建或修改的任何自定义 G 代码方言。
- **插件配置：** 已启用/已禁用的插件列表。
- **通信日志：** Rayforge 和您的激光之间发送的数据的详细记录。
- **系统信息：** 您的操作系统以及安装的 Rayforge 和关键库的版本。
- **应用程序状态：** 其他可以帮助定位错误来源的内部信息。

> **隐私说明：** 报告 **不** 包括您的任何设计文件（SVG、DXF 等）或个人操作系统数据。它只包含与 Rayforge 应用程序及其与激光连接直接相关的信息。

# Багатошаровий робочий процес

Багатошарова система Rayforge дозволяє організовувати складні завдання в
окремі етапи обробки, кожен з власними операціями та налаштуваннями. Це є
необхідним для комбінування різних процесів, таких як гравіювання та різання,
або для роботи з кількома матеріалами.

## Що таке шари?

**Шар** у Rayforge — це:

- **Контейнер** для робочих деталей (імпортовані фігури, зображення, текст)
- **Робочий процес**, що визначає, як ці деталі обробляються
- **Етап**, що обробляється послідовно під час виконання завдань

**Ключове поняття:** Шари обробляються по порядку, один за одним, що дозволяє
вам контролювати послідовність операцій.

:::note Шари та робочі деталі
Шар містить одну або кілька робочих деталей. При імпорті SVG-файлів із шарами,
кожен шар з вашого дизайну стає окремим шаром у Rayforge. Це дозволяє зберегти
організацію вашого дизайну точно так, як ви його створили.
:::

---

## Навіщо використовувати кілька шарів?

### Типові випадки використання

**1. Гравіювання, потім різання**

Найпоширеніший багатошаровий робочий процес:

- **Шар 1:** Растрове гравіювання дизайну
- **Шар 2:** Контурне різання обрису

**Навіщо окремі шари?**

- Гравіювання спочатку гарантує, що деталь не зрушить під час гравіювання
- Різання в кінці запобігає падінню деталей до завершення гравіювання
- Різні налаштування потужності/швидкості для кожної операції

**2. Багатопрохідна різання**

Для товстих матеріалів:

- **Шар 1:** Перший прохід з помірною потужністю
- **Шар 2:** Другий прохід з повною потужністю (та сама геометрія)
- **Шар 3:** Третій прохід за потреби

**Переваги:**

- Зменшує обпалювання порівняно з одним проходом на високій потужності
- Кожен шар може мати різні налаштування швидкості/потужності

**3. Проекти з кількома матеріалами**

Різні матеріали в одному завданні:

- **Шар 1:** Нарізати деталі з акрилу
- **Шар 2:** Гравіювати дерев'яні деталі
- **ШШар 3:** Маркувати металеві деталі

**Вимоги:**

- Кожен шар спрямовано на різні ділянки робочого столу
- Різна швидкість/потужність/фокусування для кожного матеріалу

**4. Імпорт шарів SVG**

Імпорт SVG-файлів з існуючою структурою шарів:

- **Шар 1:** Елементи гравіювання з SVG
- **Шар 2:** Елементи різання з SVG
- **Шар 3:** Елементи насічки з SVG

**Робочий процес:**

- Імпортувати SVG-файл, що містить шари
- Увімкнути «Використовувати оригінальні вектори» у діалозі імпорту
- Вибрати шари для імпорту зі списку виявлених шарів
- Кожен шар стає окремим шаром у Rayforge

**Вимоги:**

- Ваш SVG-файл повинен використовувати шари (створені в Inkscape або
  аналогічному ПЗ)
- Увімкнути «Використовувати оригінальні вектори» при імпорті
- Назви шарів зберігаються з вашого програмного забезпечення для дизайну

## Створення та керування шарами

### Додавання нового шару

1. **Натисніть кнопку «+»** на панелі Шарів
2. **Назвіть шар** описово (наприклад, «Шар гравіювання», «Шар різання»)
3. **Шар з'являється** у списку шарів

**Типово:** Нові документи починаються з одним шаром.

### Властивості шарів

Кожен шар має:

| Властивість           | Опис                                                   |
| --------------------- | ------------------------------------------------------ |
| **Назва**             | Назва, що відображається у списку шарів                |
| **Видимість**         | Перемикає видимість на полотні та в попередньому перегляді |
| **Робочий процес**    | Операції, що застосовуються до деталей у цьому шарі    |
| **Ротаційний**        | Чи виконується цей шар у ротаційному режимі            |
| **Робочі деталі**     | Фігури/зображення, що містяться в цьому шарі           |

### Ротаційний режим для шару

Якщо у вас налаштований [ротаційний пристрій](../machine/rotary), ви можете
увімкнути ротаційний режим для окремих шарів. Це дозволяє поєднувати роботу на
плоскій поверхні та циліндричну роботу в одному проекті — наприклад,
гравіювання дизайну на плоскій кришці коробки на одному шарі та обгортання
тексту навколо циліндричного корпусу на іншому.

Шари з активним ротаційним режимом відображають невеликий значок ротації у
списку шарів. Кожен шар запам'ятовує власне ротаційне налаштування, тому ви
можете вільно їх комбінувати.

:::note Шари як контейнери
Шари є контейнерами для ваших робочих деталей. При імпорті SVG-файлів із
шарами, кожен шар з вашого дизайну стає окремим шаром у Rayforge.
:::

### Зміна порядку шарів

**Порядок виконання = порядок шарів у списку (зверху вниз)**

Для зміни порядку:

1. **Перетягніть** шари на панелі Шарів
2. **Порядок має значення** - шари виконуються зверху вниз

**Приклад:**

```
Панель Шарів:
1. Шар гравіювання     Виконується першим
2. Шар насічки         Виконується другим
3. Шар різання         Виконується останнім (рекомендовано)
```

### Видалення шарів

1. **Виберіть шар** на панелі Шарів
2. **Натисніть кнопку видалення** або натисніть Delete
3. **Підтвердьте видалення** (всі робочі деталі шару будуть видалені)

:::warning Видалення є незворотним
Видалення шару прибирає всі його робочі деталі та налаштування робочого
процесу. Використовуйте «Скасувати» у разі випадкового видалення.
:::

## Призначення робочих деталей шарам

### Ручне призначення

1. **Імпортуйте або створіть** робочу деталь
2. **Перетягніть робочу деталь** до потрібного шару на панелі Шарів
3. **Або скористайтеся панеллю властивостей** для зміни шару деталі

### Імпорт шарів SVG

При імпорті SVG-файлів з увімкненим «Використовувати оригінальні вектори»:

1. **Увімкніть «Використовувати оригінальні вектори»** у діалозі імпорту
2. **Rayforge виявляє шари** з вашого SVG-файлу
3. **Виберіть шари** для імпорту за допомогою перемикачів
4. **Кожен вибраний шар** стає окремим шаром з власною робочою деталлю

:::note Виявлення шарів
Rayforge автоматично виявляє шари з вашого SVG-файлу. Кожен шар, створений у
вашому програмному забезпеченні для дизайну, з'явиться як окремий шар у
Rayforge.
:::

:::note Тільки імпорт векторів
Вибір шарів доступний лише при використанні прямого імпорту векторів. У режимі
трасування весь SVG обробляється як одна робоча деталь.
:::

### Переміщення робочих деталей між шарами

**Перетягування:**

- Виберіть робочу деталь(і) на полотні або панелі Документа
- Перетягніть до цільового шару на панелі Шарів

**Вирізати та вставити:**

- Виріжте робочу деталь з поточного шару (Ctrl+X)
- Виберіть цільовий шар
- Вставте (Ctrl+V)

### Діалог імпорту SVG

При імпорті SVG-файлів діалог імпорту надає параметри, що впливають на
обробку шарів:

**Режим імпорту:**

- **Використовувати оригінальні вектори:** Зберігає ваші векторні шляхи та
  структуру шарів. При увімкненні з'являється розділ «Шари», що показує всі
  шари з вашого файлу.
- **Режим трасування:** Перетворює SVG на растрове зображення та обводить
  контури. Вибір шарів у цьому режимі вимкнений.

**Розділ Шари (тільки для імпорту векторів):**

- Показує всі шари з вашого SVG-файлу
- Кожен шар має перемикач для увімкнення/вимкнення імпорту
- Назви шарів з вашого ПЗ для дизайну зберігаються
- Тільки вибрані шари імпортуються як окремі шари

:::tip Підготовка SVG-файлів для імпорту шарів
Щоб використовувати імпорт шарів SVG, створіть свій дизайн із шарами в
програмному забезпеченні, такому як Inkscape. Використовуйте панель Шарів для
організації дизайну, і Rayforge збереже цю структуру.
:::

## Робочі процеси шарів

Кожен шар має **робочий процес**, що визначає, як обробляються його робочі
деталі.

### Налаштування робочих процесів шарів

Для кожного шару ви обираєте тип операції та налаштовуєте її параметри:

**Типи операцій:**

- **Контур** - Обводить контури (для різання або насічки)
- **Растрове гравіювання** - Гравіює зображення та заповнює області
- **Гравіювання з глибиною** - Створює гравіювання зі змінною глибиною

**Додаткові можливості:**

- **Утримуючі ніжки** - Маленькі перемички для утримання деталей під час
  різання
- **Оверскан** - Продовжує розрізи за межі фігури для чистіших країв
- **Коригування керфу** - Компенсує ширину різу лазера

### Типові налаштування шарів

**Шар гравіювання:**

- Операція: Растрове гравіювання
- Налаштування: 300-500 DPI, помірна швидкість
- Зазвичай не потрібні додаткові параметри

**Шар різання:**

- Операція: Контурне різання
- Параметри: Утримуючі ніжки (для закріплення деталей), Оверскан (для чистих
  країв)
- Налаштування: Повільніша швидкість, вища потужність

**Шар насічки:**

- Операція: Контур (низька потужність, не прорізає наскрізь)
- Налаштування: Низька потужність, висока швидкість
- Призначення: Лінії згину, декоративні лінії

## Видимість шарів

Керуйте тим, які шари відображаються на полотні та в попередніх переглядах:

### Видимість на полотні

- **Значок ока** на панелі Шарів перемикає видимість
- **Приховані шари:**
  - Не відображаються на 2D-полотні
  - Не відображаються в 3D-перегляді
  - **Все одно включені в згенерований G-code**

**Випадки використання:**

- Приховувати складні шари гравіювання під час позиціювання шарів різання
- Очищати полотно під час роботи з конкретними шарами
- Зосередитися на одному шарі за раз

### Видимість проти Увімкнено

| Стан                       | Полотно | Попередній перегляд | G-code |
| -------------------------- | ------- | ------------------- | ------ |
| **Видимий та увімкнений**  | Так     | Так                 | Так    |
| **Прихований та увімкнений** | Ні    | Ні                  | Так    |
| **Видимий та вимкнений**   | Так     | Так                 | Ні     |
| **Прихований та вимкнений** | Ні     | Ні                  | Ні     |

:::note Вимкнення шарів
:::

Щоб тимчасово виключити шар із завдань без його видалення, вимкніть операцію
шару або деактивуйте його в налаштуваннях шару.

## Порядок виконання шарів

### Як обробляються шари

Під час виконання завдання Rayforge обробляє кожен шар по порядку зверху вниз.
В межах кожного шару всі робочі деталі обробляються перед переходом до
наступного шару.

### Порядок має значення

**Неправильний порядок:**

```
1. Шар різання
2. Шар гравіювання
```

**Проблема:** Нарізані деталі можуть випасти або зрушити до гравіювання!

**Правильний порядок:**

```
1. Шар гравіювання
2. Шар різання
```

**Чому:** Гравіювання відбувається, поки деталь ще закріплена, потім різання
її відокремлює.

### Кілька проходів

Для товстих матеріалів створіть кілька шарів різання:

```
1. Шар гравіювання
2. Шар різання (Прохід 1) - 50% потужності
3. Шар різання (Прохід 2) - 75% потужності
4. Шар різання (Прохід 3) - 100% потужності
```

**Порада:** Використовуйте однакову геометрію для всіх проходів різання
(дублюйте шар).

## Додаткові техніки

### Групування шарів за матеріалом

Використовуйте шари для організації за матеріалом під час виконання змішаних
завдань:

```
Матеріал 1 (Акрил 3 мм):
  - Шар гравіювання акрилу
  - Шар різання акрилу

Матеріал 2 (Фанера 3 мм):
  - Шар гравіювання дерева
  - Шар різання дерева
```

**Робочий процес:**

1. Обробити всі шари Матеріалу 1
2. Замінити матеріали
3. Обробити всі шари Матеріалу 2

**Альтернатива:** Використовуйте окремі документи для різних матеріалів.

### Пауза між шарами

Ви можете налаштувати Rayforge на паузу між шарами. Це корисно, коли вам
потрібно:

- Замінити матеріал під час завдання
- Перевірити прогрес перед продовженням
- Скоригувати фокусування для різних операцій

Для налаштування пауз між шарами скористайтеся функцією хуків у налаштуваннях
вашої машини.

### Специфічні налаштування для кожного шару

Робочий процес кожного шару може мати унікальні налаштування:

| Шар         | Операція  | Швидкість   | Потужність | Проходи |
| ----------- | --------- | ----------- | ---------- | ------- |
| Гравіювання | Растрове  | 300 мм/хв   | 20%        | 1       |
| Насічка     | Контур    | 500 мм/хв   | 10%        | 1       |
| Різання     | Контур    | 100 мм/хв   | 90%        | 2       |

## Найкращі практики

### Правила найменування

**Хороші назви шарів:**

- «Гравіювання - Логотип»
- «Різання - Зовнішній контур»
- «Насічка - Лінії згину»
- «Прохід 1 - Чорнова різка»
- «Прохід 2 - Фінальна різка»

**Погані назви шарів:**

- «Шар 1», «Шар 2» (неописові)
- Довгі описи (будьте стислими)

### Організація шарів

1. **Зверху вниз = порядок виконання**
2. **Гравіювання перед різанням** (загальне правило)
3. **Групуйте пов'язані операції** (все різання, все гравіювання)
4. **Використовуйте видимість** для фокусування на поточній роботі
5. **Видаляйте невикористані шари** для підтримання чистоти проектів

### Підготовка SVG-файлів для імпорту шарів

**Для найкращих результатів при імпорті SVG-шарів:**

1. **Використовуйте панель Шарів** у вашому ПЗ для дизайну для організації
   дизайну
2. **Надайте змістовні назви** кожному шару (наприклад, «Гравіювання»,
   «Різання»)
3. **Зберігайте шари плоскими** - уникайте розміщення шарів всередині інших
   шарів
4. **Збережіть ваш файл** та імпортуйте в Rayforge
5. **Перевірте виявлення шарів** у діалозі імпорту

Rayforge працює найкраще з SVG-файлами, створеними в Inkscape або аналогічному
векторному редакторі, що підтримує шари.

### Продуктивність

**Багато шарів:**

- Немає значного впливу на продуктивність
- 10-20 шарів є типовими для складних завдань
- Організовуйте логічно, а не для мінімізації кількості шарів

**Спрощуйте за потреби:**

- Об'єднуйте схожі операції в один шар, де це можливо
- Використовуйте менше растрових гравіювань (найбільш ресурсомісткі)

## Усунення несправностей

### Шар не генерує G-code

**Проблема:** Шар відображається в документі, але не в згенерованому G-code.

**Рішення:**

1. **Перевірте, чи шар має робочі деталі** - Порожні шари пропускаються
2. **Перевірте налаштування робочого процесу** - Шару потрібна операція
3. **Перевірте параметри операції** - Потужність > 0, коректна швидкість тощо
4. **Перевірте видимість деталей** - Приховані деталі можуть не оброблятися
5. **Перегенеруйте G-code** - Внесіть невелику зміну для примусової
   регенерації

### Неправильний порядок шарів

**Проблема:** Операції виконуються в неочікуваному порядку.

**Рішення:** Змініть порядок шарів на панелі Шарів. Пам'ятайте: верхній =
перший.

### Шари перекриваються в попередньому перегляді

**Проблема:** Кілька шарів показують перекриваючий вміст у попередньому
перегляді.

**Пояснення:** Це нормально, якщо шари спільно використовують одну ділянку XY.

**Рішення:**

- Використовуйте видимість шарів для тимчасового приховування інших шарів
- Перевірте 3D-перегляд, щоб побачити глибину/порядок
- Переконайтеся, що це навмисно (наприклад, гравіювання та різання однієї
  фігури)

### Робоча деталь у неправильному шарі

**Проблема:** Робоча деталь була призначена до неправильного шару.

**Рішення:** Перетягніть робочу деталь до правильного шару на панелі Шарів або
в дереві Документа.

### Шари SVG не виявлені

**Проблема:** Імпортується SVG-файл, але шари не з'являються у діалозі імпорту.

**Рішення:**

1. **Перевірте структуру SVG** - Відкрийте ваш файл у Inkscape або аналогічному
   ПЗ для перевірки наявності шарів
2. **Увімкніть «Використовувати оригінальні вектори»** - Вибір шарів доступний
   лише в цьому режимі імпорту
3. **Перевірте, що ваш дизайн має шари** - Переконайтеся, що ви створили шари
   у ПЗ для дизайну, а не просто групи
4. **Перевірте наявність вкладених шарів** - Шари всередині інших шарів можуть
   не виявлятися правильно
5. **Перезбережіть ваш файл** - Іноді перезбереження в актуальній версії ПЗ
   для дизайну допомагає

### Імпорт шарів SVG показує неправильний вміст

**Проблема:** Імпортований шар показує вміст з інших шарів або порожній.

**Рішення:**

1. **Перевірте вибір шарів** - Підтвердьте, що правильні шари увімкнені у
   діалозі імпорту
2. **Перевірте ваш дизайн** - Відкрийте оригінальний файл у ПЗ для дизайну,
   щоб підтвердити, що кожен шар містить правильний вміст
3. **Перевірте спільні елементи** - Елементи, що з'являються в кількох шарах,
   можуть викликати плутанину
4. **Спробуйте режим трасування** - Використовуйте режим трасування як запасний
   варіант, якщо імпорт векторів має проблеми

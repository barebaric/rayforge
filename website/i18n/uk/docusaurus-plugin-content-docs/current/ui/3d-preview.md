# 3D вид

Вікно 3D виду дозволяє візуалізувати ваші траєкторії G-коду перед
відправкою їх на машину. Ця потужна функція допомагає виявляти помилки
та перевіряти налаштування вашої роботи.

![3D вид](/screenshots/main-3d.png)

## Відкриття 3D виду

Доступ до 3D виду:

- **Меню**: Вид → 3D вид
- **Клавіатура**: <kbd>F12</kbd>
- **Після генерації G-коду**: Відкривається автоматично (налаштовується)

## Навігація

### Керування мишею

- **Обертання**: Ліва кнопка миші та перетягування
- **Панорама**: Права кнопка миші та перетягування, або середня кнопка та перетягування
- **Масштаб**: Колесо миші, або <kbd>ctrl</kbd> + ліва кнопка миші та перетягування

### Попередні налаштування виду

Швидкі кути камери:

- **Зверху** (<kbd>1</kbd>): Вид з висоти пташиного польоту
- **Спереду** (<kbd>2</kbd>): Фасад
- **Ізометричний** (<kbd>7</kbd>): 3D ізометричний вид

## Відображення робочої системи координат

3D вид візуалізує активну робочу систему координат (WCS)
інакше, ніж 2D полотно:

### Сітка та осі

- **Ізольоване відображення**: Сітка та осі з'являються так, ніби початок WCS є
  світовим початком
- **Зміщення застосовано**: Вся сітка зміщена для вирівнювання з вибраним
  зміщенням WCS
- **Мітки відносно WCS**: Координатні мітки показують позиції відносно
  початку WCS, а не початку машини

Це відображення "в ізоляції" полегшує розуміння того, де ваша робота буде
виконуватися відносно вибраної робочої системи координат, без плутанини
від абсолютної позиції машини.

### Зміна WCS

3D вид автоматично оновлюється при зміні активної WCS:
- Виберіть інший WCS з випадаючого списку панелі інструментів
- Сітка та осі зміщуються, щоб відобразити новий початок WCS
- Мітки оновлюються для показу координат відносно нового WCS

:::tip WCS в 3D виді
3D вид показує ваші траєкторії відносно вибраного WCS. Коли ви
змінюєте WCS, ви побачите, що траєкторії ніби переміщуються, тому що референсна
точка (сітка) змінилася, а не тому, що самі траєкторії перемістилися.
:::


## Параметри відображення

### Візуалізація траєкторії

Налаштуйте те, що ви бачите:

- **Показати швидкі переміщення**: Відображати переміщення (пунктирні лінії)
- **Показати робочі переміщення**: Відображати переміщення різання/гравіювання (суцільні лінії)
- **Колір за операцією**: Різні кольори для кожної операції

:::tip Кольори для кожного лазера
При використанні машин з кількома лазерними головками кожен лазер може мати
власні кольори різання та растра, налаштовані в
[Налаштуваннях лазера](../machine/laser). Це полегшує ідентифікацію того,
який лазер виконуватиме кожну операцію.
:::

### Візуалізація машини

- **Показати початок**: Відображати референсну точку (0,0)
- **Показати робочу область**: Відображати межі машини

### Налаштування якості

- **Товщина лінії**: Товщина ліній траєкторії
- **Згладжування**: Плавне рендеринг ліній (може вплинути на продуктивність)
- **Фон**: Світлий, темний або власний колір

## Контролі відтворення

Симуляція виконання роботи:

- **Відтворити/Пауза** (<kbd>пробіл</kbd>): Анімація виконання траєкторії
- **Швидкість**: Налаштувати швидкість відтворення (0.5x - 10x)
- **Крок вперед/назад**: Просунути на окремі команди G-коду
- **Перейти до позиції**: Клацнути на часовій шкалі для переходу до певної точки

### Часова шкала

Часова шкала показує:

- Поточну позицію в роботі
- Межі операцій (кольорові сегменти)
- Оціночний час в будь-якій точці

### Видимість шарів

Перемикання видимості операцій:

- Клацніть на назву операції для показу/приховування
- Зосередьтеся на конкретних операціях для інспекції
- Ізолюйте проблеми без повторної генерації G-коду

## Контрольний список перевірки

Перед відправкою на машину перевірте:

- [ ] **Траєкторія повна**: Немає відсутніх сегментів
- [ ] **В межах робочої області**: Залишається всередині меж машини
- [ ] **Правильний порядок операцій**: Гравіювання перед різанням
- [ ] **Без зіткнень**: Головка не вдаряє затискачі/кріплення
- [ ] **Правильний початок**: Починається в очікуваній позиції
- [ ] **Позиції лапок**: Утримуючі лапки в правильних місцях (якщо використовуються)

## Поради щодо продуктивності

Для великих або складних робіт:

1. **Зменшіть деталізацію ліній**: Знизьте якість відображення для швидшого рендерингу
2. **Приховайте швидкі переміщення**: Зосередьтеся тільки на робочих переміщеннях
3. **Вимкніть згладжування**: Покращує частоту кадрів
4. **Закрийте інші програми**: Звільніть ресурси GPU

## Виправлення проблем

### Перегляд порожній або чорний

- Перегенеруйте G-код (<kbd>ctrl+g</kbd>)
- Перевірте, що операції увімкнені
- Переконайтеся, що об'єкти мають призначені операції

### Повільний або затягнутий перегляд

- Зменшіть товщину ліній
- Вимкніть згладжування
- Приховайте швидкі переміщення
- Оновіть драйвери графіки

### Кольори відображаються неправильно

- Перевірте налаштування кольору за операцією
- Переконайтеся, що операції мають призначені різні кольори
- Скиньте налаштування виду до значень за замовчуванням

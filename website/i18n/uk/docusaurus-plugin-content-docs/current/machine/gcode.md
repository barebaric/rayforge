# Налаштування G-code

Сторінка "G-code" у налаштуваннях машини налаштовує, як Rayforge генерує G-code для вашої машини.

![Налаштування G-code](/screenshots/machine-gcode.png)

## Діалект G-code

Виберіть діалект G-code, який відповідає прошивці вашого контролера. Різні контролери використовують трохи різні команди та формати.

### Доступні діалекти

- **GRBL**: Найпоширеніший для хобі-лазерних різаків. Використовує M3/M5 для керування лазером.
- **Smoothieware**: Для Smoothieboard та подібних контролерів.
- **Marlin**: Для контролерів на базі Marlin.
- **GRBL-сумісний**: Для контролерів, які здебільшого слідують синтаксису GRBL.

:::інформація
Діалект впливає на те, як потужність лазера, переміщення та інші команди формуються у вихідному G-code.
:::

## Власний G-code

Ви можете налаштувати G-code, який Rayforge генерує у специфічних точках завдання.

### Початок програми

Команди G-code, що виконуються на початку кожного завдання, перед будь-якими операціями різання.

Типові використання:
- Встановлення одиниць (G21 для мм)
- Встановлення режиму позиціонування (G90 для абсолютного)
- Ініціалізація стану машини

### Кінець програми

Команди G-code, що виконуються в кінці кожного завдання, після всіх операцій різання.

Типові використання:
- Вимкнення лазера (M5)
- Повернення до початку (G0 X0 Y0)
- Паркування головки

### Зміна інструменту

Команди G-code, що виконуються при перемиканні між лазерними головками (для машин з кількома лазерами).

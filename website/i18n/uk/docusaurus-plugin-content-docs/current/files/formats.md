# Підтримувані формати файлів

Ця сторінка надає детальну інформацію про всі формати файлів, підтримувані Rayforge, включаючи можливості, обмеження та рекомендації.

## Огляд форматів

### Швидкий довідник

| Формат               | Тип    | Імпорт   | Експорт          | Рекомендоване використання           |
| -------------------- | ------- | -------- | --------------- | ------------------------- |
| **SVG**              | Вектор  | ✓ Прямий | ✓ Експорт об'єктів | Основний формат дизайну     |
| **DXF**              | Вектор  | ✓ Прямий | ✓ Експорт об'єктів | Обмін CAD           |
| **PDF**              | Змішаний   | ✓ Трасування  | –               | Експорт документів (обмежено) |
| **PNG**              | Растр  | ✓ Трасування  | –               | Фото, зображення            |
| **JPEG**             | Растр  | ✓ Трасування  | –               | Фото                    |
| **BMP**              | Растр  | ✓ Трасування  | –               | Проста графіка           |

## Векторні формати

### SVG (Scalable Vector Graphics)

**Розширення:** `.svg`
**MIME-тип:** `image/svg+xml`
**Імпорт:** Прямий парсинг векторів або трасування растру
**Експорт:** Експорт об'єктів (тільки геометрія)

**Що таке SVG?**

SVG - це векторний формат зображень на основі XML. Це **переважний формат** для імпорту дизайну в Rayforge.

**Підтримувані функції:**

- ✓ Шляхи (лінії, криві, дуги)
- ✓ Базові форми (прямокутники, кола, еліпси, полігони)
- ✓ Групи та трансформації
- ✓ Кольори обводки та заливки
- ✓ Кілька шарів
- ✓ Координатні трансформації (перенесення, обертання, масштабування)

**Непідтримувані/обмежені функції:**

- ✗ Текст (має бути конвертований у шляхи спочатку)
- ✗ Градієнти (спрощуються або ігноруються)
- ✗ Фільтри та ефекти (ігноруються)
- ✗ Маски та шляхи обрізки (можуть не працювати коректно)
- ✗ Вбудовані растрові зображення (імпортуються окремо, якщо можливо)
- ✗ Складні стилі обводки (пунктир можуть бути спрощені)
- ✗ Символи та елементи use (екземпляри можуть не оновлюватися)

**Примітки до експорту:**

При експорті деталі в SVG Rayforge експортує геометрію як векторні шляхи з:

- Тільки обводка (без заливки)
- Міліметрові одиниці
- Чорний колір обводки

**Найкращі практики:**

1. **Використовуйте формат Plain SVG** (не Inkscape SVG або інші варіанти специфічні для інструментів)
2. **Конвертуйте текст у шляхи** перед експортом
3. **Спрощуйте складні шляхи** для зменшення кількості вузлів
4. **Сплощуйте групи** коли можливо
5. **Видаляйте невикористані елементи** (напрямні, сітки, приховані шари)
6. **Встановлюйте одиниці документа** в мм (рідна одиниця Rayforge)

**Рекомендації ПЗ:**

- **Inkscape** (безкоштовний) - Відмінна підтримка SVG, рідний формат

### DXF (Drawing Exchange Format)

**Розширення:** `.dxf`
**MIME-тип:** `application/dxf`, `image/vnd.dxf`
**Імпорт:** Прямий парсинг векторів
**Експорт:** Експорт об'єктів (тільки геометрія)

**Що таке DXF?**

DXF - це формат креслень AutoCAD, широко використовується для обміну CAD.

**Підтримувані версії:**

- ✓ **R12/LT2** (рекомендовано - найкраща сумісність)
- ✓ R13, R14
- ✓ R2000 та пізніші (зазвичай працює, але R12 безпечніше)

**Підтримувані сутності:**

- ✓ Лінії (LINE)
- ✓ Полілінії (LWPOLYLINE, POLYLINE)
- ✓ Дуги (ARC)
- ✓ Кола (CIRCLE)
- ✓ Сплайни (SPLINE) - конвертуються в полілінії
- ✓ Еліпси (ELLIPSE)
- ✓ Шари

**Непідтримувані/обмежені функції:**

- ✗ 3D-сутності (використовуйте 2D-проекцію)
- ✗ Розміри та анотації (ігноруються)
- ✗ Блоки/вставки (можуть не інстанціюватися коректно)
- ✗ Складні типи ліній (спрощуються до суцільних)
- ✗ Текст (ігнорується, спочатку конвертуйте в контури)
- ✗ Штрихування (можуть бути спрощені або ігноровані)

**Примітки до експорту:**

При експорті деталі в DXF Rayforge експортує:

- Лінії як сутності LWPOLYLINE
- Дуги як сутності ARC
- Криві Безьє як сутності SPLINE
- Міліметрові одиниці (INSUNITS = 4)

### RFS (Ескіз Rayforge)

**Розширення:** `.rfs`
**MIME-тип:** `application/x-rayforge-sketch`
**Імпорт:** Прямий (деталі на основі ескізів)
**Експорт:** Експорт об'єктів (деталі на основі ескізів)

**Що таке RFS?**

RFS - це власний параметричний формат ескізів Rayforge. Він зберігає всі геометричні
елементи та параметричні обмеження, дозволяючи зберігати та спільно використовувати повністю
редаговані ескізи.

**Функції:**

- ✓ Всі геометричні елементи (лінії, дуги, кола, прямокутники тощо)
- ✓ Всі параметричні обмеження
- ✓ Розмірні значення та вирази
- ✓ Області заливки

**Коли використовувати:**

- Збереження багаторазових параметричних дизайнів
- Спільне використання редагованих ескізів з іншими користувачами Rayforge
- Архівування роботи в процесі

---

### PDF (Portable Document Format)

**Розширення:** `.pdf`
**MIME-тип:** `application/pdf`
**Імпорт:** Рендеринг у растр, потім трасування
**Експорт:** Не підтримується

**Що таке імпорт PDF?**

Rayforge може імпортувати файли PDF, спочатку растеризуючи їх, потім трасуючи у вектори.

**Процес:**

1. PDF рендериться у растрове зображення (за замовчуванням 300 DPI)
2. Растр трасується для створення векторних шляхів
3. Шляхи додаються до документа

**Обмеження:**

- **Не справжній векторний імпорт** - Навіть векторні PDF растеризуються
- **Втрата якості** від растеризації
- **Тільки перша сторінка** - Багатосторінкові PDF імпортують лише сторінку 1
- **Повільно для складних PDF** - Рендеринг та трасування займає час

**Коли використовувати:**

- Останній засіб, коли SVG/DXF недоступні
- Швидкий імпорт простих дизайнів
- Документи зі змішаним вмістом

**Кращі альтернативи:**

- **Експортуйте SVG з джерела** замість PDF
- **Використовуйте векторні формати** (SVG, DXF) коли можливо
- **Для тексту:** Експортуйте з текстом, конвертованим у контури

## Растрові формати

Всі растрові формати **імпортуються шляхом трасування** - автоматично конвертуються у векторні шляхи.

### PNG (Portable Network Graphics)

**Розширення:** `.png`
**MIME-тип:** `image/png`
**Імпорт:** Трасування у вектори
**Експорт:** Не підтримується

**Характеристики:**

- **Стиснення без втрат** - Без втрати якості
- **Підтримка прозорості** - Альфа-канал збережено
- **Добре для:** Логотипів, лінійної графіки, знімків екрану, всього, що потребує прозорості

**Якість трасування:**  (Відмінно для висококонтрастних зображень)

**Найкращі практики:**

- Використовуйте PNG для логотипів та графіки з чіткими краями
- Забезпечте високий контраст між переднім планом та фоном
- Прозорий фон працює краще, ніж білий

---

### JPEG (Joint Photographic Experts Group)

**Розширення:** `.jpg`, `.jpeg`
**MIME-тип:** `image/jpeg`
**Імпорт:** Трасування у вектори
**Експорт:** Не підтримується

**Характеристики:**

- **Стиснення з втратами** - Деяка втрата якості
- **Без прозорості** - Завжди має фон
- **Добре для:** Фото, зображень з суцільними тонами

**Якість трасування:**  (Добре для фото, але складно)

**Найкращі практики:**

- Використовуйте високоякісний JPEG (низьке стиснення)
- Збільште контраст перед імпортом
- Розгляньте попередню обробку в редакторі зображень
- Краще конвертувати в PNG спочатку, якщо можливо

---

### BMP (Bitmap)

**Розширення:** `.bmp`
**MIME-тип:** `image/bmp`
**Імпорт:** Трасування у вектори
**Експорт:** Не підтримується

**Характеристики:**

- **Без стиснення** - Великі розміри файлів
- **Простий формат** - Широко сумісний
- **Добре для:** Простої графіки, вивід старого ПЗ

**Якість трасування:**  (Добре, але не краще за PNG)

**Найкращі практики:**

- Конвертуйте в PNG для меншого розміру файлу (без різниці в якості)
- Використовуйте лише якщо джерельне ПЗ не може експортувати PNG/SVG
